# 09 — AI Excerpt Generation

## Problem

Current post excerpts are generated by `excerptFromContent()` in `src/models/post.ts` — a mechanical text truncation that strips markdown and cuts at the 160th character boundary. This produces low-quality excerpts that often break mid-sentence and miss the article's actual point.

The AI infrastructure is fully operational (multi-provider registry, DB-persisted config, AI SDK v6 integration, connection test UI), but has zero business features. The excerpt generator is the first practical application.

### Current excerpt pipeline

```
post.content → stripMarkdown() → slice(0, 160) → lastIndexOf(" ") → excerpt
```

No semantic understanding. A 3000-word technical post about React Server Components gets the same treatment as its opening paragraph about JavaScript fatigue.

## Design Decision

**Server-side AI generation triggered by explicit user action.** Not auto-generated on save.

### Principles

1. **De-AI'd output** — generated text reads like a human wrote it. No "this article explores", "in this post we discuss", or other LLM filler phrases. The prompt explicitly bans these patterns.
2. **Length control** — Chinese: 80–120 characters, English: 120–200 characters. One or two sentences max.
3. **Graceful degradation** — when AI is not configured or the call fails, the existing `excerptFromContent()` remains as fallback. The feature is additive, not a replacement.
4. **Non-blocking** — AI generation is a user-initiated action (button click), not part of the save flow. The generated text fills the textarea; the user can edit before saving.
5. **Content economy** — only the first 2000 characters of post content are sent to the LLM. Most articles establish their thesis early; sending the full article wastes tokens.

### Why slug-based API (not body-based)

The API route reads content from DB by slug rather than accepting content in the request body:

- Avoids sending the full article over HTTP when it's already in the database
- Excerpt always matches the saved version of the content
- Natural workflow: write → save → generate excerpt → save again

Trade-off: user must save the post before generating an excerpt. This is acceptable because the button only appears on the edit form (existing posts).

## Implementation

### Step 1: AI service function

**File**: `src/services/ai.ts`

Add `generateExcerpt(title, content)` function:

- Loads AI settings from DB via `getAiSettings()`
- Resolves config via existing `resolveAiConfig()` → `createAiClient()` pipeline
- Calls `generateText()` with a carefully crafted prompt
- Returns trimmed plain text
- Throws `"AI not configured"` if provider/apiKey missing

Prompt:
```
You are a blog excerpt writer. Write a concise summary for the following blog post.

Rules:
- One or two sentences, no more
- Write in the same language as the article
- Sound natural, like a human wrote it — avoid phrases like "this article discusses",
  "in this post", "the author explores"
- Capture the core insight or takeaway, not a table of contents
- Chinese: 80-120 characters. English: 120-200 characters
- No markdown formatting, no quotes, just plain text

Title: {title}

Content:
{first 2000 chars of content}
```

### Step 2: API route

**File**: `src/app/api/posts/[slug]/excerpt/route.ts` (new)

```
POST /api/posts/[slug]/excerpt
→ 200 { excerpt: "..." }
→ 400 { error: "AI provider not configured..." }  (no AI settings)
→ 404 { error: "Post not found" }
→ 502 { error: "Excerpt generation failed: ..." }  (LLM error)
```

### Step 3: Post form UI

**File**: `src/components/admin/post-form.tsx`

Add a small text button next to the "Excerpt" label:

```
Excerpt (optional, auto-generated if empty)  [✨ AI Generate]
┌─────────────────────────────────────────┐
│ Brief description of the post...        │
│                                         │
└─────────────────────────────────────────┘
```

- Button visible only when editing (`isEditing`)
- Disabled while generating (`isGenerating` state)
- On click: POST to `/api/posts/{slug}/excerpt`, fill textarea with result
- On error: alert with error message
